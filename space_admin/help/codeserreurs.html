Codes d'erreurs du logiciel et pistes de solutions

011 : Impossible d'ouvrir la connexion au serveur MySQL [xxxxxx] avec l'utilisateur [xxxxxx].

Les codes d'accès à la base de données (nom du serveur, login et mot de passe) ne sont pas présents dans le fichier _config/BD-...-connect.inc.php ou ne sont pas corrects.
Ajoutez ces codes ou corrigez-les.

012 : La base [xxxxx] n'est pas accessible sur le serveur MySQL.

Le nom de la base de données n'est par reconnu par le serveur MYSQL. Le nom peut êtes incorrect ou la base n'est pas encore créée sur le serveur.
Vérifier le nom de la base et sa présence. Bien souvent vous pouvez utiliser l'utilitaire PHPMyAdmin pour vérifier ces informations.

021 : Ce programme nécessite au moins la version X.Y.Z de PHP.

La version du module PHP sur le serveur n'est pas adaptée pour le programme. Certaines fonctions indispensables ne sont pas présentes.
Mettez à jour le serveur ou demander à votre hébergeur de le faire.

022 : Ce programme nécessite au moins la version X.Y.Z de MySQL.

La version du moteur de base de donnée MySQL du serveur n'est pas adaptée pour le programme. Certaines fonctions indispensables ne sont pas présentes.
Mettez à jour le serveur ou demander à votre hébergeur de le faire.

031 : Répertoire de téléchargement "_upload" inaccessible ou inexistant.

Le logiciel doit disposer d'un répertoire où les fichiers CSV chargés seront stockés pendant le temps nécessaire à leur traitement. En principe, ce répertoire est placé en "actes/admin/_upload". Les droits d'accès effectés à ce répertoire doivent permettre au serveur web de déplacer un fichier vers ce répertoire.
Vérifier la présence du répertoire et ses droits d'accès. Au besoin, ExpoActes peut utiliser un autre répertoire dont le chemin d'accès doit êtes précisé via le paramètre UPLOAD_DIR.

032 : Impossible de créer un fichier dans "_upload".

Le répertoire pour le téléchargement existe mais il est impossible - pour le serveur web - d'écrire un fichier dans ce répertoire.
En général ce problème est du au fait que les droits d'accès au répertoire sont insuffisant. Pour les modifier, il faut utiliser votre logiciel FTP et afficher le répertoire "admin". Les permissions attribuées au répertoire doivent êtes "drwxrwxrwx" ou encore "777". Si ce n'est pas le cas modifiez ces droits de la façon suivante : Sur le nom du répertoire "_upload" faites un clic droit et sélectionner l'item "Propriétés..." ou "Attributs..." puis cochez tous les droits d'accès.

033 : Impossible de ranger le fichier dans "_upload".

Le fichier qui vient d'êtes téléchargé n'a pas pu êtes stocké dans le répertoire _upload prévu à cet effet. En général, la solution est la mêtesme que pour l'erreur 032 ou éventuellement 031.

034 : Répertoire de backup "_backup" inaccessible ou inexistant.

Le repertoire _backup doit êtes créé et accessible à la racine du site (au mêtesme niveau que _config). S'il n'existe pas, il faut le créer manuellement.

035 : Impossible de créer un fichier dans "_backup".

Le répertoire pour le backup existe mais il est impossible - pour le serveur web - d'écrire un fichier dans ce répertoire.
En général ce problème est du au fait que les droits d'accès au répertoire sont insuffisant. Pour les modifier, il faut utiliser votre logiciel FTP et afficher le répertoire "admin". Les permissions attribuées au répertoire doivent êtes "drwxrwxrwx" ou encore "777". Si ce n'est pas le cas modifiez ces droits de la façon suivante : Sur le nom du répertoire "_backup" faites un clic droit et sélectionner l'item "Propriétés..." ou "Attributs..." puis cochez tous les droits d'accès.

041 : Impossible de trouver le script SQL "xxxxx.sql".

Le fichier de script ne se trouve pas dans le répertoire attendu (en principe le répertoire "install". Ré-exécuter l'installation ou la mise à jour.

051 : L'envoi du mail via la procédure interne à PHP n'a pas réussi.

Si l'adresse du destinataire est correctement spécifiée, la cause la plus probable est que le module d'envoi de courrier électronique interne à PHP n'est soit pas disponible ou n'a pas été configuré correctement. C'est souvent le cas lors de l'utilisation en local avec EasyPHP. Dans le cas d'un serveur hébergé, il est aussi possible que l'hébergeur ait décidé de ne pas offrir ce service.
Pour résoudre la problème, il faut soit corriger le paramétrage du module PHP (fichier php.ini) soit utiliser le module d'envoi de mail intégré à ExpoActes en configurant celui-ci de manière adéquate.

052 : Paramètres de gestion du mail incomlètement configurés.

Les paramètres de la focntion interne de mail doivent êtes complétés. Au minimum, il faut préciser le nom du serveur SMTP, le nom de la machine qui envoie le mail et l'adresse de l'expéditeur, plus, parfois, le login et le mot de passe.

053 : Impossible de se connecter au serveur mail "xxxxx".

Le serveur spécifié n'existe pas, n'est pas accessible ou ne founit pas le service SMTP sur le port 25.

054 : Problème lors du dialogue avec le serveur mail "xxxxx" : xxxxxx.

Le serveur SMTP a renvoyé le message cité. Il est possible que ce serveur requière une authentification (vérifier le login et le mot de passe) ou qu'il ne soit pas accessible pour le moment.

061 : Librairie GD indisponible.

La librairie graphique GD2 n'a pas été activée sur votre hébergement. Celle-ci est indispensable pour générer à la volée les codes "captcha" : codes masqués de protection des formulaire d'auto-inscription. Si vous avez accès au paramétrage PHP, il suffit bien souvent de retirer le point virgule devant la ligne ";extension=php_gd2.dll" du fichier PHP.INI.

Si cela est inpossible avec votre hébergement vous pouvez aussi désactiver la gestion des codes captcha dans le paramétrage des utilisateurs.

Pour la modification des paramètres du logiciel, voir la configuration avancée.
